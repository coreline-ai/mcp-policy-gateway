# implement_20260703_132853.md

작성 일시: `2026-07-03 13:28:53 KST`

이 문서는 사용자가 target MCP를 Gateway 밖에 직접 등록해 보호 경계를 우회하는 상황을 줄이기 위한 다음 개발 범위를 고정한다. 목표는 "사용자 PC를 강제로 잠그는 기능"이 아니라, 현재 repo가 할 수 있는 **client config 검증, 사용 흐름 고정, managed deployment claim boundary 문서화**를 구현하는 것이다.

## 개발 목적

MCP Runtime Policy Gateway의 보호는 target MCP가 client에 직접 등록되지 않고 Gateway 뒤에 있을 때만 성립한다. 따라서 다음 패치는 사용자가 Claude Desktop, Codex CLI, generic MCP client 설정에 `mcp-policy-gateway` 외 target MCP를 직접 추가했는지 감지하고, 운영 문서에서 self-managed / validated-local / managed-enforced 배포 모드를 명확히 구분한다.

완료 후 사용자는 다음 흐름으로 점검할 수 있어야 한다.

```bash
npm run config:client -- claude-desktop
npm run config:validate -- claude-desktop /path/to/client-config.json
npm run smoke:mcp-client-preflight
```

검증 통과 의미는 "현재 설정 파일에는 Gateway 외 MCP server가 없다"이다. 이것이 OS/MDM 수준의 강제 차단이나 모든 MCP client 자동 보호를 뜻하지는 않는다.
