## 문서 기반 제약 사항
- 이 프로젝트의 정본 목적은 `MCP Runtime Policy Gateway`다. 일반 proxy, scanner, dashboard, data collector로 범위를 넓히지 않는다.
- 강한 enforcement 주장은 `MCP Client -> Gateway -> Target MCP` 배치에서만 가능하다.
- stdio MVP의 tenant/client/actor는 config single principal이다. per-user/team enforcement를 구현된 것처럼 쓰지 않는다.
- target credential은 policy YAML, command args, env dump, audit/evidence, tool schema에 원문 저장하지 않는다.
- output redaction은 best-effort다. 완전 DLP, exfiltration 완전 차단, OS 격리 제공을 주장하지 않는다.
- target 등록은 privileged operator/config action이다. runtime user input으로 임의 process spawn이 열리면 안 된다.
- MCP 프로토콜 처리와 transport는 기존 MCP SDK와 표준 라이브러리를 우선 사용한다.

## Phase 1. Fail-Closed Policy And Snapshot Gates

### 목표
- policy matching에서 catch-all rule이 누락되어 block/approval rule이 무시되는 fail-open 경로를 제거한다.
- target snapshot이 call 직전에 stale/incomplete로 바뀌면 target forwarding 전에 fail-closed한다.
- default-deny, hidden-tool block, changed-tool default-deny가 같은 call gate를 통과하도록 정리한다.

### 책임 단위
- Policy rule matcher: `match.any`, target/tool pattern, rule priority, default effect를 단일 책임으로 검증한다.
- Policy schema guard: unsupported effect와 invalid matcher가 조용히 무시되지 않게 로딩 단계에서 실패시킨다.
- Snapshot call gate: target 호출 직전의 callability 판단을 gateway/session boundary에 둔다.

### 구현 태스크
- [x] `PolicyEngine`의 matched rule 계산에서 `match.any`를 명시적으로 포함한다.
- [x] `default: allow` + `match.any` block rule이 모든 tool call을 block하는 regression test를 먼저 추가한다.
- [x] explicit block/approval rule이 broad allow보다 우선되는지 policy priority test를 추가한다.
- [x] policy YAML에서 지원하지 않는 effect나 matcher가 있으면 startup/config load에서 실패하도록 validation을 보강한다.
- [x] `GatewayRuntime.enforce()`와 `TargetSessionManager.callTool()` 사이의 snapshot stale race를 재현하는 테스트를 작성한다.
- [x] session open, target initialize, `notifications/tools/list_changed` 이후 target call 직전에 callability를 재확인하는 gate를 추가한다.

## Phase 6. PlayMCP Full-Inventory Pre-Use Assessment Tests

### 목표
- PlayMCP 전체 MCP를 Gateway의 사전평가 대상으로 삼고, 사용자가 MCP를 연결하기 전에 "사용 가능/승인 필요/비추천/차단/수동검토" 결정을 볼 수 있게 하는 테스트 기준을 고정한다.

- 이 Phase는 "모든 MCP가 안전하다"를 증명하지 않는다. Gateway가 전체 MCP 후보를 일관된 기준으로 분류하고, 위험 도구를 기본 허용하지 않는지를 검증한다.

### 책임 단위
- Inventory loader: PlayMCP snapshot CSV/JSON을 읽고 row 누락, 중복, 필수 필드 결손을 검증한다.

- Decision mapper: 위험 라벨을 Gateway policy decision 후보(`allow`, `limited_alias`, `approval_required`, `block`, `manual_review`)로 매핑한다.
- Assessment report: MCP별 판정, 근거 라벨, 대표 위험 tool, 추천 사용자 행동을 사람이 읽는 표와 기계가 읽는 JSON/CSV로 남긴다.
