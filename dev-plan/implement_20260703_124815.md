## 개발 목적
일반 사용자가 Claude, Codex CLI, Desktop 앱의 프롬프트 창에서 `mcp-policy-gateway`를 실제로 연결하고, target MCP를 직접 추가하기 전에 Gateway MCP로 사전검증을 수행한 뒤, 결과를 운영자 등록/정책 검토로 넘길 수 있는 end-to-end 사용자 시나리오를 완성한다.

직전 계획(`implement_20260703_112129.md`)에서 PlayMCP preflight 도구 3개는 구현 완료됐다. 이번 개발은 그 다음 단계인 "처음 사용자 연결", "실제 MCP protocol 검증", "구조화된 operator handoff", "unknown/freshness 안내"를 닫는다.

## 개발 범위
- MCP client별 Gateway 등록 설정을 생성하는 repo-owned TypeScript 스크립트를 추가한다.

## 문서 기반 제약 사항
- 이 프로젝트의 정본 목적은 `MCP Runtime Policy Gateway`다.
- 보호 경로는 `MCP Client -> Gateway -> Target MCP`이며, target MCP를 client에 직접 등록하면 Gateway 보호 경로가 깨진다.
- target registration은 privileged operator/config boundary다.
- 이번 개발은 사용자가 연결 전 판단을 묻고, 운영자에게 등록 검토를 넘기는 경로를 강화한다.
- preflight는 inventory 기반 정적 사전검증이다. 실제 target MCP의 최신 동작이나 악성 여부를 보증하지 않는다.
- MCP protocol 검증에는 기존 `@modelcontextprotocol/sdk`를 우선 사용하고 새 의존성은 추가하지 않는다.
