# implement_20260703_130821.md

작성 일시: `2026-07-03 13:08:21 KST`

이 문서는 현재 구현된 MCP Runtime Policy Gateway를 기준으로 문서 정합성을 복구하고, 이후 개발자가 같은 혼선을 반복하지 않도록 회귀 검증까지 고정하기 위한 구현 계획이다.

## 개발 목적

사용자가 Claude, Codex CLI, desktop MCP client의 프롬프트 창에서 **먼저 Gateway MCP를 연결**하고, 특정 target MCP를 직접 연결하기 전에 `gateway_search_playmcp`, `gateway_preflight_mcp`, `gateway_explain_mcp_risk`로 사용 가능 여부 판단 지원을 받는 현재 제품 흐름을 문서 전반에 정확히 반영한다.

핵심 목표는 다음과 같다.

- README, handoff, dev-plan이 현재 코드의 tool surface, 검증 명령, 사용자 시나리오를 같은 방식으로 설명하게 한다.
- 과거 MVP/bootstrap 문서가 현재 완료 상태처럼 오해되지 않게 역사 문서와 현재 상태 문서를 분리한다.
- `package.json`, `src/upstream/tools.ts`, `src/upstream/gateway.ts`의 실제 동작과 문서 예시가 어긋나지 않게 한다.
- 문서 정합성 자체를 테스트로 고정해 이후 구현이 문서를 다시 낡게 만들지 않도록 한다.

## 현재 구현 기준 사실

- Default client surface는 `gateway_health`, `gateway_search_playmcp`, `gateway_preflight_mcp`, `gateway_explain_mcp_risk`, `gateway_call_tool`, `gateway_list_exposed_tools`, `gateway_request_approval`이다.
- Operator surface는 client surface 전체에 `gateway_list_targets`, `gateway_inspect_target`, `gateway_rescan_target`, `gateway_diff_target`, `gateway_get_audit_event`가 추가된다.
- `gateway_list_exposed_tools`는 현재 input을 받지 않으며, 전체 policy-filtered alias surface를 반환한다.
- `gateway_inspect_target` input은 `targetId`만 요구한다.
- `verify:mvp`는 `typecheck`, `test`, `smoke:preuse`, `smoke:mcp-client-preflight`, `assessment:playmcp`, `demo:mvp`, `npm audit --omit=dev`를 포함한다.
- PlayMCP 평가는 inventory 기반 static assessment이며 실제 remote MCP `tools/call`을 자동 호출하지 않는다.
- HTTP target adapter는 ADR-017 기반 guarded post-MVP surface다. true socket-IP pinning과 full remote HTTP MCP E2E는 후속이다.
