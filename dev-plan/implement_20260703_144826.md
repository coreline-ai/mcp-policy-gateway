이 문서는 P1-P3 잔여 항목을 현재 프로젝트 목적 기준으로 다시 선별하고, 실제 제품 완성도에 필요한 다음 개발 범위를 고정하기 위한 작업 문서다.

## 개발 목적

MCP Runtime Policy Gateway는 사용자가 target MCP를 직접 연결하기 전에 `tools/list`와 tool surface를 평가해 사용 가능 여부 판단을 돕고, 실제 사용 시 Gateway 뒤에서 정책을 강제하는 제품이다.

현재 stdio MVP, PlayMCP 정적 사전검증, 사용자 프롬프트 흐름, Gateway-only config 검증은 구현되어 있다. 이번 계획의 목적은 남은 P1-P3 목록 중 **코드/테스트/운영 산출물로 지금 검증 가능한 필수 항목**만 다음 개발 범위로 묶고, 범위를 흐리는 항목은 명확히 보류하는 것이다.

## 개발 범위

이번 개발계획에 포함되는 범위는 다음 네 묶음이다.

1. **Managed deployment artifact hardening**
   - `managed-enforced` 환경에서 사용자가 target MCP를 Gateway 밖에 직접 등록하지 못하게 운영자가 적용할 수 있는 배포 산출물을 정리한다.
   - repo는 MDM/GPO/read-only config 자체를 구현하지 않고, 예시 프로파일/템플릿/검증 절차/운영 체크리스트를 제공한다.
   - `config:validate`는 계속 drift detection으로 표현한다.

2. **HTTP true socket-IP pinning**
   - 현재 `assertEgressAllowed`의 DNS 검증 결과와 실제 HTTP 연결을 분리하지 않도록 transport hook을 추가한다.
   - `undici` Agent/Dispatcher 또는 Node 표준 fetch가 제공하는 공식 확장점을 먼저 검토한다.
   - 연결은 검증된 public IP로 수행하되 TLS SNI/Host header는 원래 hostname을 유지한다.
   - redirect, DNS rebinding, IPv4/IPv6, mixed DNS 결과를 fail-closed로 검증한다.

3. **HTTP MCP end-to-end 검증**
   - 기본 테스트는 repo-local Streamable HTTP MCP fixture로 결정론적으로 수행한다.
   - 실제 remote MCP 서버 대상 테스트는 `REMOTE_MCP_E2E_URL` 같은 환경 변수가 있을 때만 실행하는 optional smoke로 둔다.
   - 자동 `tools/call`은 side effect가 없는 fixture/known-safe tool에 한정한다.

4. **PlayMCP inventory refresh 운영**
   - 외부 crawler를 만들기보다, 새 snapshot CSV를 repo-owned validator/import script로 검증하고 교체 가능한 운영 흐름을 만든다.
   - row 수, 필수 컬럼, id uniqueness, toolCount/toolNames 정합성, category coverage, snapshot date/freshness note를 검증한다.
   - 새 snapshot이 고위험 MCP를 기본 `usable`로 떨어뜨리지 않는지 테스트한다.

## Phase 3. HTTP True Socket-IP Pinning 구현

### 목표

ADR-017의 잔여 리스크인 "DNS 검증 후 실제 연결이 다른 IP로 갈 수 있는 문제"를 transport hook으로 줄인다.

### 구현 태스크

- [ ] Node/undici 공식 확장점을 확인하고 pinning 구현 방식을 결정한다.
  - [ ] `undici`가 필요하면 dependency 추가 이유를 기록한다.
  - [ ] 새 dependency 없이 가능하면 기존 dependency set을 유지한다.
- [ ] `assertEgressAllowed` 결과를 connection layer에 전달할 contract를 정의한다.
  - [ ] hostname
  - [ ] allowed resolved addresses
  - [ ] chosen address 또는 lookup result
- [ ] `HttpTargetAdapter`가 guarded fetch에 pinned dispatcher/agent를 사용하도록 수정한다.
- [ ] TLS SNI/Host는 원래 URL hostname을 유지한다.
- [ ] redirect destination마다 shape validation, DNS validation, pinning을 다시 적용한다.
- [ ] `GATEWAY_EGRESS_ALLOW_PRIVATE=true`는 local dev/test escape hatch로만 유지한다.
- [ ] pinning 실패, lookup mismatch, empty DNS, mixed public/private DNS를 fail-closed 처리한다.

### 자체 테스트

- [ ] injected resolver가 public IP를 반환하면 해당 IP로 pinning되는지 unit test로 확인한다.
- [ ] resolver가 public 검증 후 private connect candidate를 반환하는 rebinding 케이스가 차단되는지 확인한다.
- [ ] private/metadata/loopback/link-local/ULA/CGNAT/mapped IPv4가 계속 차단되는지 기존 phase6 테스트가 통과한다.
- [ ] redirect to metadata가 차단되는지 확인한다.
- [ ] `GATEWAY_EGRESS_ALLOW_PRIVATE=true`에서만 local fixture가 허용되는지 확인한다.

### 이슈 및 수정

- [ ] 발견 이슈 없음

### 완료 조건
