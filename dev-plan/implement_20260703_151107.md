# implement_20260703_151107.md

작성 일시: `2026-07-03 15:11:07 KST`

이 문서는 후순위로 분류했던 항목 중 현재 repo 안에서 바로 구현 가능한 항목만 골라 별도 개발 범위로 고정한다.

## 개발 목적

아래 후순위 항목 중 외부 인증 체계, OS 정책, 별도 UI 제품 설계, 공식 live API 접근 없이 바로 구현 가능한 것은 **License / CI / release automation 중 CI와 release hygiene 자동화**다.

이번 개발의 목적은 공개 배포나 main branch 보호 전에 필요한 반복 검증, release checklist, 문서 claim guard를 repo-owned 산출물로 만드는 것이다. 단, `LICENSE` 본문은 법적/오너 결정이 필요하므로 임의로 선택하지 않는다.

## 즉시 구현 가능성 판정

| 항목 | 바로 구현 가능 여부 | 이번 계획 반영 | 판정 이유 |
|---|---:|---:|---|
| per-user/team auth | 아니오 | 제외 | stdio MVP는 config single principal이다. authenticated transport/control plane 설계가 선행되어야 한다. |
| OS/process sandbox | 아니오 | 제외 | Gateway 코드가 아니라 OS/container/network policy 영역이다. 지금은 문서 가이드까지만 가능하지만 실제 sandbox 구현으로 잡지 않는다. |
| Dashboard UI | 부분 가능, 지금은 제외 | 제외 | 정적 report UI는 가능하지만 승인/감사/정책 운영 dashboard는 API, 권한, UX 범위가 커진다. 현재 CLI/MCP tool/HTML report로 MVP 사용 가능하다. |
| License / CI / release automation | 예, 단 LICENSE 선택 제외 | 포함 | `.github/workflows`가 없고 `verify:mvp`는 이미 있어 CI 연결이 즉시 가능하다. release checklist/runbook도 repo 문서로 바로 만들 수 있다. LICENSE 종류는 오너 결정 필요. |
| PlayMCP live crawler | 아니오 | 제외 | 공식 접근 경로, 인증, rate limit, 데이터 사용 조건이 확인되어야 한다. 현재는 snapshot refresh 운영까지만 안전하다. |

## 개발 범위

- GitHub Actions 기반 검증 workflow 추가
  - main/pull_request에서 `npm ci`, `npm run verify:mvp` 실행

  - generated report artifact 업로드 여부 결정
- release checklist 문서 추가
  - version/tag/release note 기준
  - `npm run verify:mvp` 통과 기준
  - PlayMCP report 재생성 기준
  - 보안/비밀/금지 claim 점검 기준
  - LICENSE 선택 gate
- release runbook 또는 maintainer checklist 추가
  - 배포 전 확인
  - 실패 시 rollback/재시도 기준
  - GitHub release 작성 기준

- 문서 정합성 테스트가 release hygiene 문서를 잊지 않도록 필요한 최소 테스트 추가 검토
