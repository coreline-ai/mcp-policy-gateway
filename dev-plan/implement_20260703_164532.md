# implement_20260703_164532.md

작성 일시: `2026-07-03 16:45:32 KST`

이 문서는 `public-preflight` 제품 모드를 PlayMCP에 등록 가능한 상태로 만들기 위한 **인바운드 Streamable HTTP MCP 서버(ADR-018)** 구현 범위를 고정한다. 라이브 PlayMCP 사용 가이드/심사 정책 확인 결과를 설계 제약으로 반영한다.

## 개발 목적

`PROJECT_DIRECTION §1`이 1차 제품으로 지정한 것은 PlayMCP/Kakao에 등록되는 public Remote MCP server인 **Hosted Preflight MCP**다. 평가 엔진(`src/assessment/*`), 3개 public tool 로직, 등록 메타데이터 문안, claim 경계는 이미 구현/문서화되어 있으나, **외부에서 도달 가능한 HTTPS `/mcp` 인바운드 서버 자체가 없다.** 현재 업스트림은 stdio 전용이다(`grep` 기준 `StreamableHTTPServerTransport`/`/mcp`/`.listen()` 부재).

이번 개발의 목적은 ADR-018과 `docs/playmcp-public-hosted-preflight.md`의 endpoint contract를 실제 코드로 구현하고, PlayMCP 심사 정책이 요구하는 하드닝(응답 크기 상한, PII 미수집, origin/rate 제어)을 fail-closed로 배선해 **임시 등록 → AI Chat 테스트 → 심사 요청** 흐름에 진입 가능한 상태를 만드는 것이다.

runtime-gateway(stdio, target proxy) 모드는 별개 배포 모드로 유지하며, 이번 범위는 target을 spawn/등록/호출하지 않는 preflight-only 인바운드 서버로 한정한다.

## 우선순위 티어 (심사 통과 기준)

같은 이번 범위 안에서도 PlayMCP 심사 통과에 직결되는 항목과 표준 위생 항목을 구분한다. 구현은 P0를 먼저 완결한 뒤 P1을 채운다.

| 티어 | 항목 | 근거 |
|---|---|---|
| **P0 (심사 직결)** | `/mcp` transport, 3-tool allowlist + target 미호출, **24k 응답 상한**, **입력 PII 거부**, 등록 네이밍/데이터출처 준수, HTTPS 종단(운영자 선행) | 미충족 시 자동 반려 또는 등록 불가 |
| **P1 (표준 위생)** | `/healthz`, Origin 검증, body/query 상한, rate limit | 운영 안정성·abuse 대응. 임시 등록 테스트는 통과 가능하나 공개 전 필요 |

## 개발 범위

이번 개발계획에 포함되는 범위는 다음 다섯 묶음이다.

1. **인바운드 Streamable HTTP transport core**
   - Node `http` 서버 + MCP SDK `StreamableHTTPServerTransport`로 `/mcp` 구현. 신규 의존성은 SDK 표준 확장점으로 부족할 때만 추가하고 이유를 기록한다.
   - `POST /mcp`(JSON-RPC 1건/요청), `GET /mcp`(SSE 미지원 시 405), `DELETE /mcp`(stateless → 405).
   - notification/response 수신 시 202, `MCP-Protocol-Version` 미지원 값 400, `Accept: application/json, text/event-stream` 처리.
   - 기본 stateless(`Mcp-Session-Id` 미발급). 미래에 발급하면 미검증/만료 세션 fail-closed.

2. **public-preflight tool allowlist + fail-closed**
   - `GATEWAY_PUBLIC_MODE=public-preflight`에서 `tools/list`가 `gateway_search_playmcp`/`gateway_preflight_mcp`/`gateway_explain_mcp_risk` 3개만 반환.
   - `gateway_call_tool`/approval/audit/rescan/diff/target registry/operator/dynamic alias는 인바운드 표면에서 unknown-tool 또는 fail-closed. target spawn/등록/호출 경로가 인바운드 모드에서 배선되지 않음을 코드로 보장.
   - 기존 `gatewayToolsForMode`/`toolSurfaceMode`를 재사용하고 새 mode를 최소 추가한다.

3. **Hosted 하드닝**
   - `/healthz`(MCP tool 아님) — tenant/private target 메타데이터 미노출.
   - `Origin` 존재 시 검증, `GATEWAY_ALLOWED_ORIGINS` 미부합 거부.
   - `GATEWAY_MAX_BODY_BYTES` body 상한, `GATEWAY_MAX_QUERY_CHARS` free-text 질의 상한.
   - `GATEWAY_RATE_LIMIT_WINDOW_MS`/`_MAX` 기본 abuse throttle(in-memory, 단일 인스턴스 기준으로 문서화).
   - **응답 크기 상한**: 3개 tool 응답 직렬화 결과가 상한(기본 20k자, 24k 반려선 아래) 초과 시 candidates 축소/텍스트 절단 + `truncated: true` 표기.
   - **입력 PII 가드**: query/reasonForUse/declaredTools에서 한국 주민번호·카드·계좌·여권·운전면허·외국인등록번호 패턴 탐지 시 **기본 거부(fail-closed)** — preflight 질의에 PII가 정당히 필요한 경우가 없으므로 조용한 redaction보다 명시적 거부를 기본으로 한다. 원문 미저장, 로그는 redacted category/status만.

4. **설정/배선**
   - `GATEWAY_UPSTREAM_TRANSPORT=stdio|streamable-http` 스위치를 `load-config`에 추가, 기본 stdio(기존 동작 불변).
   - `GATEWAY_HOST`/`GATEWAY_PORT`/`GATEWAY_MCP_PATH`/`GATEWAY_HEALTH_PATH` 도입. `GATEWAY_PUBLIC_BASE_URL`은 서버 동작에 불필요하므로 코드가 아닌 등록 안내 문서에만 둔다.
   - `.env.example`에 신규 변수 문서화.
   - `index.ts`는 transport 스위치로 stdio 서버 또는 HTTP 서버를 기동한다.

5. **등록 준비 산출물 / 스모크**
   - `scripts/smoke-hosted-preflight.ts` — 실제 HTTP `/mcp`로 initialize/tools/list(3개)/각 tool call/`/healthz`/차단 케이스(runtime tool·과대응답·PII 입력·비허용 origin)를 관통 검증, exit code로 통과 판정.
   - `docs/playmcp-public-hosted-preflight.md`의 Acceptance Checklist를 실제 스모크 결과와 연결.
   - 등록 메타데이터(이름/설명/starter prompt)가 심사 네이밍 규칙에 부합함을 문서로 확인.

## Phase 2. 인바운드 Streamable HTTP Transport Core

### 목표
`/mcp`에서 MCP Streamable HTTP 요청을 처리하는 인바운드 서버를 구현한다.

### 구현 태스크
- [ ] MCP SDK `StreamableHTTPServerTransport`(server 측) 사용 가능 여부와 사용법을 확인한다.
- [ ] `src/upstream/http-server.ts`에 Node `http.createServer` 기반 서버를 구현한다.
  - [ ] `POST /mcp`: JSON-RPC 1건 처리, `Accept: application/json, text/event-stream` 수용, JSON 또는 SSE 응답.
  - [ ] JSON-RPC notification/response 수신 → 202, body 없음.
  - [ ] `GET /mcp`: SSE 미지원 시 405.
  - [ ] `DELETE /mcp`: 405(stateless).
  - [ ] `MCP-Protocol-Version` 미지원/무효 → 400.
- [ ] 기존 `GatewayRuntime`/tool dispatch를 재사용해 요청을 처리한다(공개 모드 제한은 Phase 3).
- [ ] **응답 방식 확인**: `POST /mcp`가 `application/json` 단일 응답만으로도 PlayMCP 콘솔 information-load와 AI Chat 호출을 통과하는지 검증한다(SSE 강제 여부 불명 → JSON 우선, SSE는 SDK 기본 지원 범위로 둠).
- [ ] graceful shutdown(SIGINT/SIGTERM)로 서버/세션 정리.

### 자체 테스트
- [ ] loopback bind 후 `initialize` → `tools/list` → `tools/call` 왕복이 성공한다.
- [ ] GET/DELETE가 405를 반환한다.
- [ ] 잘못된 protocol version이 400을 반환한다.
