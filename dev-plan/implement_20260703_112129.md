## 개발 목적
일반 사용자가 Claude, Codex CLI, Desktop 앱 같은 MCP client의 프롬프트 창에서 "OO MCP를 써도 돼?"라고 물었을 때, Gateway MCP가 PlayMCP inventory 기반 정적 사전검증 결과를 조회해 사용 가능 여부 판단 지원, 위험 근거, Gateway 정책 권장, 다음 행동을 답할 수 있게 한다.

현재 프로젝트는 runtime policy gateway와 PlayMCP 187개 정적 보고서 생성은 갖췄지만, 일반 사용자가 프롬프트 창에서 직접 호출할 수 있는 사전검증 MCP tool surface는 아직 없다. 이번 개발은 이 간극을 메우는 client-facing 기능을 추가한다.

## 개발 범위
- PlayMCP 187개 inventory row를 기존 repo-owned TypeScript 평가 로직으로 조회하는 user-facing preflight service를 추가한다.
- MCP client surface에 읽기 전용 Gateway 도구를 노출한다.
  - `gateway_search_playmcp`: 사용자가 말한 MCP 이름/키워드로 후보를 찾는다.
  - `gateway_preflight_mcp`: 특정 MCP의 decision, risk label, 대표 위험 tool, 권장 Gateway 정책, 사용자 다음 행동을 반환한다.
  - `gateway_explain_mcp_risk`: risk label과 decision 이유를 일반 사용자 문장으로 설명한다.
- README에 "프롬프트 창에서 시작하는 실사용 흐름"을 추가한다.
- client mode에서 위 도구가 노출되고, operator-only target registration 경계를 넘지 않는지 테스트로 고정한다.
- 금지 표현 없이 "사용 가능 여부 판단 지원", "정적 사전검증", "Gateway 정책 권장"으로만 설명한다.

## 문서 기반 제약 사항
- 이 프로젝트의 정본 목적은 `MCP Runtime Policy Gateway`다. 일반 scanner, marketplace, dashboard, data collector로 범위를 넓히지 않는다.
- client에는 Gateway만 등록하고, target MCP 등록은 privileged operator/config 경계에 둔다.
- 이번 preflight 도구는 사용자가 target MCP를 직접 연결하기 전에 판단을 돕는 읽기 전용 도구다.
- preflight 결과는 inventory 기반 정적 평가다. 실제 remote MCP 서버의 최신 상태나 악성 여부를 보증하지 않는다.
- high-risk label이 붙은 MCP는 기본 `usable`로 떨어지면 안 된다.
- 기존 `src/assessment/*` 평가 로직을 재사용하고, one-off Python 또는 보고서 HTML 파싱으로 구현하지 않는다.

## 최소 책임 단위
- Inventory source resolver: 기본 `DEFAULT_PLAYMCP_INVENTORY_PATH`와 선택적 `PLAYMCP_INVENTORY_CSV`를 결정한다.
- Preflight service: inventory row를 로드하고 기존 `buildAssessmentReport`/`assessRow` 결과에서 검색, 단건 조회, 후보 ranking을 수행한다.
- Preflight presenter: decision/risk label을 일반 사용자 문장, Gateway 권장 정책, operator handoff 문장으로 변환한다.
- Upstream tool adapter: MCP tool input schema와 `handleToolCall` dispatch만 담당한다.
- Documentation: Claude/Codex/Desktop 프롬프트에서 어떤 질문을 하고 어떤 답을 받는지 설명한다.

## 사용자 기준 목표 흐름
| 단계 | 사용자 행동 | Gateway 기대 동작 |
|---|---|---|
| 1 | Claude/Codex/Desktop 프롬프트를 연다. | MCP client에 등록된 `mcp-policy-gateway`가 client surface 도구를 제공한다. |
| 2 | "카카오맵 MCP 연결해도 돼?"라고 묻는다. | client가 `gateway_preflight_mcp` 또는 `gateway_search_playmcp`를 호출할 수 있다. |
| 3 | 후보가 여러 개면 사용자가 하나를 고른다. | Gateway는 상위 후보와 confidence/reason을 반환한다. |
| 4 | 특정 MCP가 선택된다. | Gateway는 decision, risk labels, 대표 위험 tool, 권장 정책, 다음 행동을 반환한다. |
| 5 | 사용자는 직접 연결하지 않고 운영자/설정 단계로 넘긴다. | 결과에 operator handoff가 포함되고, target registration은 기존 operator 경계에 남는다. |
