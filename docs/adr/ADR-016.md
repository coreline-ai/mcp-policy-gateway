# ADR-016: Use Underscore Tool Name Grammar (No Dots)

- Status: Accepted
- Date: 2026-07-01

## Context

V2 verified against real Claude Code: after mcp__<server>__<tool> namespacing, a dot violates the API tool-name constraint ^[a-zA-Z0-9_-]{1,128}$.

## Decision

All exposed names use [a-z0-9_] only. Admin tools use gateway_* (e.g. gateway_list_targets); target aliases use <target_slug>__<tool_slug>. No dots anywhere.

## Consequences

Names are API-safe without relying on client-specific sanitization; enforced by a test.
