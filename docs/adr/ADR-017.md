# ADR-017: HTTP Target Adapter Behind An SSRF Egress Guard

- Status: Accepted
- Date: 2026-07-02

## Context

Streamable HTTP targets (post-MVP, ADR-006/T10/Q14) let a registered `endpoint_url` point at internal metadata services (169.254.169.254), private networks, or hosts that DNS-rebind to private IPs after validation — classic SSRF.

## Decision

`HttpTargetAdapter` connects over the MCP Streamable HTTP transport, and every connection passes an egress guard: scheme allowlist (default `https`), optional host allowlist, and a block on loopback/private/link-local/ULA/CGNAT/metadata IPs — both for literal-IP hosts and for every DNS-resolved address. Registration validates the URL shape synchronously; the transport is given a guarded `fetch` that re-validates egress on every request (DNS-rebinding mitigation). Private egress is permitted only via an explicit `allowPrivate` policy flag for local dev.

## Consequences

HTTP targets are usable without opening an SSRF hole. Residual: true socket-IP pinning (connect to the validated address) needs an undici `Agent` lookup hook — deferred; the current per-request re-validation is the dependency-free mitigation. Full HTTP end-to-end needs a remote MCP server, so it is deferred to integration; the guard, registration, and adapter routing are unit-tested.
